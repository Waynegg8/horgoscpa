# 開發任務 25：客戶詳情 API

**對應規格：** 後端 `docs/開發指南/後端/客戶管理-後端規格.md`

**對應手冊：** `docs/使用手冊/功能模組-02-客戶管理.md`

## 範圍

實作 `GET /api/v1/clients/:id` API，返回單一客戶的完整詳細資料。

## 輸入

- `client_id`（路徑參數）

## 輸出

- 客戶基本資料（統一編號、公司名稱、負責人員、聯絡方式）
- 客戶標籤陣列
- 客戶備註、收款備註
- 建立/更新時間

## 驗證與權限

- 需登入
- 員工可查看所有客戶（小型事務所彈性設計）
- 客戶不存在或已刪除回傳 404

## 實作重點

- 使用 JOIN 查詢負責人員名稱
- 使用 GROUP_CONCAT 或子查詢獲取標籤陣列
- 遵守 JSON Envelope 格式
- 回傳完整的客戶資料供前端顯示

## 驗收標準

- [ ] API 回傳正確的客戶詳細資料
- [ ] 包含標籤陣列與負責人員名稱
- [ ] 404 處理正確
- [ ] 已部署並可透過前端呼叫
